# /workflow-mgr - 워크플로우 관리

반복 작업의 워크플로우를 분석하고 생성/수정합니다.

## 1. 니즈 분석 (Analyze Needs)
- 새로운 반복 작업이 발견되거나, 기존 절차의 개선이 필요한지 분석합니다.
- **소스**: `Issue.md` (이슈 패턴), `git log` (커밋 패턴), 또는 사용자 요청.

## 2. 워크플로우 생성/수정 (Create or Modify)
- `.claude/commands/` 폴더에 `.md` 파일을 생성하거나 수정합니다.
- **언어**: 반드시 **한국어**로 작성합니다.
- **형식**: 명확한 단계(Step)와 실행 가능한 명령어(Code Block)로 구성합니다.

## 3. 고도화 및 표준화 (Refine & Standardize)
- **참조 명시**: 관련된 룰 파일(`.claude/rules/`)을 명시적으로 언급합니다.

## 4. 검증 (Verify)
- **논리 검증**: 작성된 워크플로우가 논리적으로 맞는지, 경로는 정확한지 확인합니다.
- **동작 검증**: 실제 워크플로우를 실행하여 의도대로 동작하는지 확인합니다.

## 5. 참조 (References)
- 스킬(Skill) 관련 작업은 `/skill-mgr` 워크플로우를 사용합니다.
